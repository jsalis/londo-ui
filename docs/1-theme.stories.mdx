# Theme

The theme object can be accessed from anywhere in the component tree with the `useTheme` hook.

```js
import { useTheme } from "styled-components";
```

### Space

`theme.space`

| Key | Alias  | Value   |
| --- | ------ | ------- |
| `0` | _none_ | `0px`   |
| `1` | `"xs"` | `4px`   |
| `2` | `"sm"` | `8px`   |
| `3` | `"md"` | `16px`  |
| `4` | `"lg"` | `32px`  |
| `5` | _none_ | `64px`  |
| `6` | _none_ | `128px` |
| `7` | _none_ | `256px` |

-   Numbers from 0 to 7 and alias strings are converted to values on the space scale.
-   Numbers greater than 7 are converted to raw pixel values.
-   Negative values can be used for negative margins.
-   String values are passed as raw CSS values.
-   Array values are converted into [responsive values](#breakpoints).

### Breakpoints

`theme.breakpoints`

| Key    | Value    |
| ------ | -------- |
| `"md"` | `768px`  |
| `"lg"` | `1200px` |

Breakpoints are CSS lengths intended for use in media queries. The breakpoint scale is used to create mobile-first
responsive media queries based on an array of values. **All** Styled System component props will create media queries
for its corresponding CSS properties when defined as an array of values.

For example, using a `margin` value of `[0, 1, 2]` creates styles with multiple mobile-first min-width media queries.

```jsx harmony
<Box margin={[0, 1, 2]} />
```

```css
.Box-hash {
    margin: 0;
}

@media screen and (min-width: 768px) {
    .Box-hash {
        margin: 4px;
    }
}

@media screen and (min-width: 1200px) {
    .Box-hash {
        margin: 8px;
    }
}
```

To skip certain breakpoints, you can pass `null` to any position in the array to avoid generating unnecessary CSS.

```jsx harmony
<Box width={[1, null, 2]} />
```
